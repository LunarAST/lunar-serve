# lunar-serve

**Read-only HTTP distribution layer for LunarAST ecosystem data.**

`lunar-serve` provides a lightweight, zero-dependency HTTP interface for consuming `lunar-map.json` — the global topology generated by `lunar map`. It serves both human-readable Markdown summaries and structured JSON, with parameterized filtering for AI agents and dashboards.

## Quick Start

```bash
# Generate topology data first
lunar map -o /opt/lunar-map.json

# Start the server
lunar-serve /opt/lunar-map.json
```

Server listens on `http://0.0.0.0:8787` by default. Set `LUNAR_SERVE_PORT` to override.

## Endpoints

### Ecosystem Summary

| Endpoint | Description |
|:---|:---|
| `GET /lunar-map.md?summary=true` | Ecosystem summary (~200 tokens, ideal for AI first contact) |
| `GET /lunar-map.md` | Full topology in Markdown |
| `GET /lunar-map.md?scope=<project>` | Single project scoped view |
| `GET /lunar-map.md?status=<orphaned\|unused>` | Filter alignments by status |
| `GET /lunar-map.md?path=<path>` | Filter alignments by path substring |
| `GET /lunar-map.md?style=mermaid` | Mermaid topology diagram |
| `GET /lunar-map.json` | Full topology JSON |

### GitHub Mirror URLs

Projects registered in `repos.json` with a `github` source type are accessible via URLs mirroring their GitHub paths:

```
GET /{owner}/{repo}/tree/{branch}
```

Example: `https://lunar.aifify.com/Jasonmilk/Anaphase-Helix/tree/rs`

This makes the URL immediately recognizable to both humans and AI agents.

### Unified API

All projects are also accessible via:

```
GET /api/v1/projects/{name}/map.md
```

### Legacy Compatibility

```
GET /project/{name}
```

### Health Check

```
GET /healthz → 200 OK
```

## Configuration

`lunar-serve` loads `repos.json` from the same directory as `lunar-map.json` to build GitHub mirror mappings and validate project identities.

### repos.json Format

```json
{
  "version": "0.5.0",
  "projects": [
    {
      "name": "cellrix",
      "displayName": "Cellrix",
      "source": {
        "type": "github",
        "github": {
          "owner": "your-github-org",
          "repo": "cellrix",
          "branch": "main"
        }
      },
      "visibility": "public"
    },
    {
      "name": "my-internal-service",
      "displayName": "Internal Service",
      "source": { "type": "local" },
      "visibility": "private"
    },
    "my-library"
  ]
}
```

Entries can be:
- **Simple string** (default: local project, public visibility)
- **Detailed object** with `source` (github or local), `displayName`, and `visibility`

## Relationship with Other Components

| Component | Relationship |
|:---|:---|
| `lunar` CLI | Produces `lunar-map.json` consumed by lunar-serve |
| `lunar-scope` | Can consume `lunar-map.json` from lunar-serve in online mode |
| RouteAST Sub-Protocol | Defines the alignment statuses rendered in summaries |

`lunar-serve` is purely a read-only data distributor. It never writes to disk or modifies configuration files.
